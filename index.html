Environment Nuance

Mobile vs. Web (Key Differences)

Operational differences that govern how we launch updates and how users consume new features.

Web Platforms

Access Method URL via standard internet browsers (Safari, Chrome).
Release Speed Instant. Updates take effect immediately upon deployment.
User Adoption Automatic. All users are forced to load the latest code on reload.
Hardware Access Limited capability to utilize physical device hardware.

Mobile Application

Access Method Downloadable setup via Apple App Store & Google Play Store.
Release Speed Delayed. Apple/Google reviews require 2 hours to 2 days.
User Adoption Gradual. Users must download the package updates manually or automatically.
Hardware Access Deep access (push notifications, cameras, biometrics, offline cache).

Our Feature Lifecycle

How we collaborate from the initial idea document down to deploying live code on production.

graph TD A[1. PD Team - Feature PRD] --> B[2. UI/UX Team - Designs] B --> C[3. PD Review & Approval] C --> D[4. Tech Lead - Technical Planning & Jira] D --> E[5. Tech Team - Dev Phase] E --> F[6. Step 1-3
PRD & Design Stage
PD drafts feature expectations; UI/UX creates visual flows which are approved by PD.
Step 4-5
Jira & Dev Phase
Tech Lead breaks PRD into task boards. Developers estimate timelines and start coding.
Step 6
QA Verification Loop
QA tests logic on staging. If bugs are found, they go to Jira for dev fixes and retest.
Step 7-8
UAT & Deploy
PD runs User Acceptance Testing (UAT) on Staging. Once approved, code is released to Live users.
Tech Strength

Technical Capabilities (BD Pitch Kit)

How our technical systems deliver business value and high performance for our brands and creators.

Flutter Cross-Platform

Single-Codebase Mobile Engineering

Utilizes a unified structure to write application files that build for iOS and Android simultaneously.

Business Impact: Launches features on both app stores at the exact same time, providing an identical experience.
NestJS Modular Core

Enterprise-Grade Server Framework

Built with highly scalable patterns (Domain-Driven Design) ensuring predictable application routing.

Business Impact: Protects servers from crashing during high-volume campaign traffic hours.
PostgreSQL & Prisma

Type-Safe Relational Storage

Prisma ORM abstracts sql operations into strictly typed models mapping database relations reliably.

Business Impact: Guarantees bank-grade security for user data, verification files, and transaction ledgers.
Redis & BullMQ

Non-blocking Worker Queues

Buffers large surges of webhook notifications and automations in memory without blocking server threads.

Business Impact: Scales Instagram DM campaigns to millions of messages without slowing down core interfaces.
Socket.IO Real-time

Continuous WebSocket Handshakes

Establishes continuous duplex channels between user devices and servers for messaging streams.

Business Impact: Delivers chats instantly in real-time, removing manual refresh delays.
Razorpay & Cashfree

Multi-Gateway Payment Foundations

Integrates premium SDK systems for automated bank settlement and secure transactions.

Business Impact: Automates instant influencer wallet payouts and provides brands with seamless pay options.
SigNoz Observability

Active Metrics & Tracing

OpenTelemetry exporter continuously profiles system speeds, logging operations, and database latency markers.

Business Impact: Triggers immediate alerts if any request slows by 200ms, enabling pre-emptive fixes.

Safe Deployments

Isolated Environments

We run completely separate environment infrastructures to guarantee reliability for active campaigns.

Staging Environment

Pre-Production Testing

A duplicate replica of our production platform used for initial regression verification.

Purpose
Validation of new features, bug fixes, and rigorous automated testing loops.
Why it matters
Allows PD to perform User Acceptance Testing (UAT) and BD to preview features safely without breaking active campaigns.
Production Environment

Live Creator Operations

The active user environment delivering real influencer marketing campaigns and transactions.

Purpose
Runs production integrations, real-time message automation queues, and transaction audits.
Strict Rule
No code is ever uploaded directly to Production without passing QA testing and securing official PD sign-off on Staging.
Team Alignment

Collaboration Best Practices

Guidelines to establish highly efficient operational bridges between our teams and developers.

Tips for the PD Team

Define Clear Edge-Cases

Outline scenario rules inside the PRDs (e.g., "What happens if KYC uploads fail?" or "What actions occur during Razorpay failures?").

Maintain Clean Jira Boards

Link visual Figma boards and PRD specification links directly in primary Jira tickets to establish a single source of truth.

Complete Prompt UAT Audits

Test feature deployments on Staging within 24 to 48 hours once QA flags the status as 'Ready for UAT' to prevent release bottlenecks.

Tips for the BD Team

Build Quality Feedback Channels

Document exact details when reporting user problems (e.g., screenshot samples, client details, and specific physical device models).

Channel Requests Through PD

Discuss new client request ideas with PD first so they are strategically sorted and prioritized on our feature roadmaps.

Coordinate Data Requests via Tech Lead

For custom analytical reports not yet in the Admin Panel, coordinate with the Tech Lead to schedule SQL database query extracts safely.
