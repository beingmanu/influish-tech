Overview & Features

The Influish Backend is a scalable, strictly-typed API service engineered on top of the progressive **NestJS** framework. It acts as the transaction processor and data orchestrator for both our Flutter mobile client and React dashboard panels.

Our backend engine handles several enterprise-grade operational workflows:

  • Authentication & Guard Gates: Combines NestJS passport JWT validation routes with direct multi-role firebase admin scopes checking (Brand, Influencer, Admin).
  • Payment Clearances: Handles Razorpay and Cashfree checkout sessions, atomic database wallet locks, and webhook clearing logic.
  • Real-time Socket Rooms: Low-latency chat routing built with Socket.IO multiplexing layers.
  • Asynchronous Messaging: Instagram DM triggers and push notifications processes decoupled via non-blocking BullMQ/Redis worker pipelines.

Technical Stack Mappings

To maintain type safety down to the database schema level, we run strict Node package dependencies. Here is our core technology layout:

Domain Layer Core Dependency Packages Functional Usage
Server Engine @nestjs/core, typescript, express Module dependency injections, DTO validations, and HTTP routing controllers.
Database Mapper prisma, @prisma/client SQL migration compiles, models mappings, and database queries abstraction.
Task Queue bullmq, ioredis Instagram webhook buffers, push alerts distribution, and chron jobs triggers.
Integrations nestjs-razorpay, cashfree-pg, openai, @aws-sdk/client-s3 Clearing bank gateways, AI prompts, and S3 KYC files storage.
Observability @opentelemetry/api, winston, swagger-ui-express SigNoz custom telemetry traces, structured logs, and auto-generated API pages.

Database Setup & Migrations

Our Postgres schema mapping is driven entirely by Prisma. Developers pull fresh updates and seed models through these simple CLI sequences:

1. Generate Local Prisma Types

Run this command to build typescript queries abstractions based on current models:

Terminal (Bash) 
npx prisma generate

2. Execute Database Migrations

Apply database changes locally or on development servers:

Terminal (Bash) 
npx prisma migrate deploy

3. Start Development Hot-Reload

Launch NestJS application watch compilation processes:

Terminal (Bash) 
npm run start:dev
Interactive API Playground
Once NestJS boots successfully, access the Swagger API playground to execute query routes directly at: http://localhost:3000/api

Source Directory Structure

The code modules are housed under `/src`, organized strictly by domains. Below is the folder mapping:

src/
├── main.ts // Bootstrap server, setups guards, starts OpenTelemetry
├── app.module.ts // Root modules managing domain loading
├── auth/ // JWT logins and Firebase synchronizations
├── brand/ // Brand operations and profile configurations
├── influencer/ // Influencer profiles and KYC validation queues
├── campaign/ // Campaigns CRUD operations and cron handlers
├── chat/ // low-latency Socket.IO gateway hubs
├── wallet/ // Atomic transactional ledgers and payouts
├── notification/ // Background FCM alert integrations
└── instagram/ // BullMQ asynchronous Instagram DM automation hooks
  • Controllers: Standardize HTTP endpoint entries and call service layers.
  • Services: House business computations, entity validations, and database writes.
  • Modules: Decouple configurations and load required domain providers.

Git Commit Workflow

To enforce clean code quality before integrations, we set up **Husky** hooks on all local repositories. These hooks run automatic checks at specific git workflow stages:

Pre-commit Hook

Triggered before commits compile. Runs Lint-staged to auto-fix styling errors and blocks commits if typescript compiler warnings exist.

Commit-msg Hook

Enforces the Conventional Commit standard structure: type(scope): description. Messages violating this will be blocked.

Conventional Commit Examples:

Commit Command Purpose
feat(auth): add google signin Introducing a brand new auth endpoint feature.
fix(wallet): secure Razorpay webhook Resolving a transaction validation bug.
docs(readme): add docker setup Modifying system orientation documentation files.
refactor(chat): simplify socket auth Restructuring code logic for cleaner execution.

Monitoring & Observability

Our active telemetry layer maps system latencies continuously, logging metric pipelines to prevent service bottlenecks:

  • Winston Structured Logging: Auto-formats server actions and error dumps, streaming structured log variables directly to analytics collectors.
  • OpenTelemetry Instrumentation: Automatically profiles database operations speeds and slow endpoints.
  • SigNoz Dashboard Portal: Aggregates system metrics, highlighting latencies crossing limits (e.g. database transactions taking >200ms).
